# RunAs Tool

Цей проєкт - портативна утиліта для виконання консольних команд від імені адміністратора без необхідності вводити пароль адміністратора на очах у кінцевого користувача. Конфігурація надійно шифрується (за допомогою симетричного алгоритму та `PBKDF2`) і зберігається у локальному файлі `.enc`.

## Як це працює

Програма працює у двох режимах:

1. **Режим створення конфігу (GUI):**
   При запуску `main.py` (або скомпільованого `RunAsTool.exe`) без параметрів відкривається зручне графічне вікно, де ви можете:
   - Ввести ім'я адміністратора (або іншого користувача з підвищеними привілеями).
   - Ввести пароль адміністратора (цей пароль буде зашифровано).
   - Вказати список команд з будь-якими необхідними параметрами.
   - **Задати майстер-пароль** (пароль для шифрування файлу).
   - Зберегти конфігурацію у зашифрований файл `.enc`.

2. **Режим виконання:**
   Щоб запустити команди, ви або кінцевий користувач повинні відкрити файл `.enc` через програму (наприклад, `RunAsTool.exe config.enc` або перетягнувши файл на іконку програми). 
   - Програма попросить ввести **майстер-пароль** для розшифрування.
   - Після успішного введення майстер-пароля, програма розшифровує дані та виконує вказані команди через `CreateProcessWithLogonW` з підвищеними привілеями.
   - Під час виконання показується невелике вікно прогресу.

## Технічні деталі та Безпека

- **Портативність:** Файл `.enc` не прив'язаний до комп'ютера чи системи. Ви можете переносити його на флешці разом з `RunAsTool.exe` і запускати на будь-якому ПК за умови знання майстер-пароля.
- **Шифрування:** Пароль адміністратора шифрується надійним потоковим шифром (генерація потоку через `SHA-256`).
- **Генерація ключа:** Ключ шифрування генерується динамічно з вашого **майстер-пароля** за стандартом `PBKDF2` (100 000 ітерацій з випадковою "сіллю"), що робить файл стійким до підбору паролів (brute-force). Програма не має вшитих універсальних ключів.
- **Легкість:** Не вимагає жодних важких сторонніх бібліотек (використовує стандартні модулі `tkinter`, `ctypes`, `hashlib`), завдяки чому скомпільований файл `exe` має дуже малий розмір.

## Збірка

Щоб зібрати `.exe` файл:
1. Переконайтесь, що ви запустили `_START_.bat` для ініціалізації середовища.
2. Запустіть `_BUILD_exe_.bat`.
3. Готовий автономний файл `RunAsTool.exe` з'явиться у папці `dist/`.
